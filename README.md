# 📚 Online Book Store – SQL Data Analysis Project

## 🚀 Project Overview

Designed and analyzed a relational database for an Online Book Store to uncover customer purchasing behavior, sales trends, inventory status, and revenue performance. Leveraged SQL to transform raw transactional data into actionable business insights that support data-driven decision-making.

## 🛠️ Skills & Technologies

* SQL (PostgreSQL)
* Database Design & Normalization
* Data Analysis
* Data Modeling
* Business Intelligence Concepts

## 🔑 Key SQL Concepts Applied

* Designed normalized database schemas with Primary Keys, Foreign Keys, and Constraints.
* Performed data extraction and filtering using WHERE, IN, LIKE, BETWEEN, and CASE statements.
* Utilized aggregate functions (SUM, AVG, COUNT, MAX, MIN) for sales and customer analysis.
* Applied JOIN operations to combine data across Customers, Orders, Books, and Authors tables.
* Implemented Subqueries and Common Table Expressions (CTEs) for complex analytical queries.
* Used Window Functions (ROW_NUMBER, RANK, DENSE_RANK) to identify top-performing books and customers.
* Conducted inventory analysis to monitor stock levels and identify restocking requirements.

## 📊 Business Insights Generated

* Identified the Top Best-Selling Books based on sales volume.
* Analyzed Top Customers contributing the highest revenue.
* Tracked Monthly and Yearly Revenue Trends.
* Evaluated Most Popular Book Genres and Categories.
* Monitored Inventory Levels and Low-Stock Products.
* Measured Order Volume and Customer Purchase Patterns.

## 🎯 Project Impact

* Enabled data-driven decision-making through sales and customer analytics.
* Improved visibility into inventory management and restocking needs.
* Provided actionable insights into customer preferences and revenue drivers.
* Demonstrated advanced SQL problem-solving and analytical capabilities using real-world business scenarios.
